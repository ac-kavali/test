# <span class="color-purple">Makefile</span>

>Make file in python is used like automation system or shortcuts platform to build the project:

- Install Dependencies.
- Create/Remove/Initial the virtual Environment.
- Generate the start point files (map, data, assets)
- Install Packages
- Or even format the code with a specific code standard like flake8

### <span class="color-cyan">How to use it</span>
```python
make run
```
**To install Dependencies**
```python
make install
make run-venv
```
**Clean Files**
```css
make clean
```
**remove everything (venv)**
```css
make fclean
```

# <span class="color-purple">Make For This Project</span>

### <span class="color-cyan">1) make install</span>
>_Should prepare the environment, so your code can run_

**What should do exactly**
- install dependencies (flake8, mypy, maybe colorama, etc.)
- usually using: pip install -r requirements

### <span class="color-cyan">2) make run</span>
>_Run your program **Normally**_

- Execute main script with Python.
- With a default/specific config file.

### <span class="color-cyan">3) make Debug</span>
>_Run your program but inside a debugger_

**Using Python built-in debugger (pdb)**
_Example_
```css
python3 -m pdb a_maze_ing.py config.txt
```
Then you can step through code (step...step), inspect variables, etc.

### <span class="color-cyan">4) make clean</span>
>Remove All garbage/cache files generated by python tools

**What should delete**
- __pycache__/
- *.pyc
- .mypy_cache/
- .pytest_cache/ (if you use pytest)
- .ruff_cache/ (if you ever use ruff)

**_Note:_**
clean = "remove temporary python-generated junk files"

### <span class="color-cyan">5) make lint</span>
==The subject explicitly demands this command:==
>Used to check the code style and format/show rules to be followed.

It must run:
```js
flake8 .
mypy . --warn-return-any --warn-unused-ignores --ignore-missing-imports \
       --disallow-untyped-defs --check-untyped-defs=
```

### <span class="color-cyan">6) make lint-strict</span>
Runs:
```js
flake8 .
mypy . --strict
```
_lint-strict = "hard mode mypy check"_
